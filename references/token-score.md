# Token 消耗：10 分制

Token 消耗是 `Skill × 任务场景` 的估算，不是 Skill 的固定属性。0 表示很省，10 表示极高。共用计算和报告规则见 [scoring-rubric.md](scoring-rubric.md)。

## 先写基准任务

“好用吗”中的 Token 分数必须基于这个 Skill 的**典型核心任务**，不能基于一个它根本无法完成的用户任务。至少写清：输入资料数量、目标长度或页数、图片/媒体数量、联网来源数量、修改与复核轮次、是否使用多个 AI agent。

- 用户需求匹配：用用户的真实任务作为基准；
- 用户需求不匹配：用官方定位中的典型核心任务作为基准，再在“适合吗”说明当前需求不会进入这条执行路径；
- 禁止把“Skill 没有干活，只做了一次检查”算成低 Token，从而制造便宜好用的错觉；
- 正文默认报告推荐路径。

## 五个维度

| 消耗来源 | 最高分 | 高分情形 |
|---|---:|---|
| 初始上下文 | 2.0 | Skill、参考资料和用户输入合计超过约 20k tokens |
| 推理与往返 | 3.0 | 多个 AI agent、反复规划、搜索或超过 4 轮修改 |
| 工具返回内容 | 2.0 | 大量搜索结果、日志或工具输出回到上下文 |
| 生成与复核 | 2.0 | 长报告、多版本、逐页或逐项复核 |
| 大文件与多媒体 | 1.0 | 多文件、大量图片、OCR、音视频或大数据 |

## 等级

| 分数 | 等级 |
|---:|---|
| 0—2.0 | 低 |
| 2.1—4.5 | 中等 |
| 4.6—7.0 | 高 |
| 7.1—10 | 很高 |

## 两本账

若 Skill 调用外部模型，分别写：

1. 当前 AI agent 的 Token：加载 Skill、读取工具输出、分析和生成；
2. 外部模型用量：API、子服务或其他模型独立产生的计费。

本地脚本计算不直接产生模型 Token。图片、搜索、存储等非 Token API 费用列入“额外涉及费用”，不混入 Token 分。

## 报告要求

- 给出可关闭的高消耗环节；
- 分开写当前 AI agent 的 Token 与外部模型用量。
