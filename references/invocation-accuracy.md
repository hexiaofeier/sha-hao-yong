# Skill 调用准确性

本项根据目标 Skill 的名称、description、触发词、适用范围和排除范围，判断 AI agent 是否容易在合适的任务中调用它。

## 检查项

1. 名称是否能提示它解决什么问题；
2. description 是否同时写清能力和使用场景；
3. 是否覆盖用户会自然说出的常见表达，而不只依赖内部术语；
4. 触发条件是否把对象、动作和任务意图写清；
5. 是否声明不适用范围，避免看到宽泛关键词就抢任务；
6. 正文规则是否与 description 一致。

## 定性结果

| 结果 | 判断 |
|---|---|
| 较准确 | 该调用时容易命中，不相关任务也有明确排除范围 |
| 偏宽 | 关键词或对象范围过大，可能在不需要时被调用 |
| 偏窄 | 依赖少量固定词或内部术语，用户换一种说法就可能漏调用 |
| 无法判断 | 缺少 description、触发说明或可读取的规则文件 |

## 输出要求

- 报告写结果、判断依据和最主要的漏调用或误调用风险；
- 说明栏直接写“根据名称、描述和触发规则判断”。
