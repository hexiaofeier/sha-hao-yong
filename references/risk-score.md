# 风险值：10 分制

风险值评估安装和使用候选对象时会接触多少本地改动、数据、权限和外部操作。0 表示很少，10 表示很多。共用计算和报告规则见 [scoring-rubric.md](scoring-rubric.md)。

## 五个维度

| 风险来源 | 最高分 | 低分情形 | 高分情形 |
|---|---:|---|---|
| 本地代码与系统改动 | 2.0 | 纯文本、只读 | 管理员权限、系统服务或广泛磁盘改动 |
| 数据与隐私 | 2.0 | 公开查询词 | Cookie、邮件、私有库、敏感数据或持续同步 |
| 凭证与权限 | 2.0 | 无凭证、公开只读 | 发布、删除、交易、管理员或破坏性权限 |
| 依赖与供应链 | 2.0 | 无依赖或成熟依赖 | 下载执行来源不清、混淆代码、未知二进制 |
| 不可逆与自动外部操作 | 2.0 | 只生成草稿 | 自动发布、发送、删除、交易或大范围外部操作 |

可选高风险分支单列。

## 等级

| 分数 | 等级 |
|---:|---|
| 0—1.5 | 低 |
| 1.6—3.5 | 较低 |
| 3.6—5.5 | 中等 |
| 5.6—7.5 | 高 |
| 7.6—10 | 很高 |

## 红灯

发现混淆代码、未知二进制、凭证外传、管理员/交易权限或无确认的大范围删除时，直接列红灯并建议暂停安装。

## 报告要求

- 内容准确性、科学性和交付质量放在“能不能完成”，不塞进风险分。
