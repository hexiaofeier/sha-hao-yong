# 任务复杂度：10 分制

任务复杂度描述 Skill 已经可用后，完成一次典型核心任务需要多少用户投入。0 表示给出简单输入后即可直接得到结果，10 表示需要多轮准备、选择、人工处理和故障恢复。共用计算和报告规则见 [scoring-rubric.md](scoring-rubric.md)。

它不计算首次安装、账号注册、Key 获取、环境配置和持续维护；这些属于安装复杂度。

## 五个维度

| 维度 | 最高分 | 评分内容 |
|---|---:|---|
| 输入准备 | 2.0 | 输入数量、格式要求、资料整理与前置判断 |
| 操作轮次 | 2.0 | 从调用到交付需要多少轮指令、确认和工具切换 |
| 分支与选择 | 2.0 | 是否需要选择模式、数据源、路径、工具或处理方案 |
| 人工介入 | 2.0 | 是否需要复制粘贴、补数据、核验、编辑或手工接力 |
| 失败恢复 | 2.0 | 错误是否清楚，能否自动降级、从中间继续或快速重试 |

## 评分原则

- 以最常见的典型核心任务为基准，不按功能全开的极端路径评分；
- 步骤多但能够自动连续完成、进度清楚且无需用户判断，不等于复杂；
- 方法内部有很多思考步骤，但用户只需一次输入和一次确认时，不把内部推理轮次都算成操作轮次；
- 用户必须反复补充材料、切换工具或判断错误时，增加分数；
- 未实测时只根据说明文件和工作流推测。

## 等级

| 分数 | 等级 | 普通人解释 |
|---:|---|---|
| 0—1.0 | 很简单 | 给出简单输入后基本自动完成 |
| 1.1—2.5 | 简单 | 少量确认，过程清楚 |
| 2.6—4.5 | 中等 | 需要准备资料或完成几次选择 |
| 4.6—6.5 | 复杂 | 多轮操作、人工核验或明显分支 |
| 6.6—8.0 | 很复杂 | 经常补数据、切工具、返工或处理失败 |
| 8.1—10 | 专业流程 | 需要熟练操作、持续判断和多人或多工具接力 |

## 报告要求

- 不把“文档写得长”直接判成任务复杂；
- 不把 AI agent 内部自动完成的步骤写成用户负担；
- 不与安装复杂度重复计算。
