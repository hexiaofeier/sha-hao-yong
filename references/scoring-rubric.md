# 定量评分共用规则

本文件适用于安装复杂度、任务复杂度、Token 消耗、风险值和推荐程度。各指标评什么、怎样分级，仍以对应 Reference 为准。

## 计算

- 先写清评分对象和基准路径，不拿可选的极端功能代替典型用法；
- 各维度按最高分的 `0%、25%、50%、75%、100%` 取值，中间状态可保留一位小数；
- 核心、推荐和全开路径差异明显时，分别评分或给出区间；
- 未确认项不补 0，写已知区间或 `未验证`；
- 只看公开文件时写“根据公开文件估算”，实际运行后才写“试跑估算”或“实测”。

## 报告

- 用户报告只显示总分、等级、基准任务或路径、主要原因，不显示维度分项；
- 内部过程文件保留分项计算，便于复核；
- 费用、Token、风险、质量和推荐程度分别判断，不互相替代；
- 分数越低或越高只表示该指标本身，不自动推出“好用”“安全”或“值得安装”。
