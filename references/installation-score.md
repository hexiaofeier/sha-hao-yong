# 安装复杂度：10 分制

安装复杂度只描述首次安装、配置和保持环境可用的负担。0 表示几乎无需配置，10 表示接近采用一套系统。共用计算和报告规则见 [scoring-rubric.md](scoring-rubric.md)。

## 五个维度

| 维度 | 最高分 | 评分内容 |
|---|---:|---|
| 本地安装与运行环境 | 3.0 | 安装单元 1.2；依赖难度 0.8；本地服务/状态 0.6；下载与硬件 0.4 |
| 账号、Key 与权限 | 2.5 | 凭证数量 0.8；获取难度 0.6；权限范围 0.7；登录态维护 0.4 |
| 外部服务依赖 | 2.0 | 必需服务数量 0.8；可替代性 0.4；数据传输 0.4；额度与可用性 0.4 |
| 持续维护与环境恢复 | 1.5 | 依赖更新 0.4；凭证续期 0.3；服务与版本变化 0.4；恢复运行环境 0.4 |
| 平台兼容与迁移 | 1.0 | 系统差异 0.3；AI agent 绑定 0.3；卸载迁移 0.2；管理员配置 0.2 |

补充判断：

- 安装单元只数用户要单独安装和维护的顶层软件，不数包管理器自动拉取的内部包；
- 当前模式不需要的远程服务不计分；可选功能在扩展路径另算；
- 调用后的任务步骤、分支、人工核验和返工不计入本表，统一进入任务复杂度；
- 只有为了让环境恢复可运行而进行的依赖修复、凭证续期和版本调整计入本表；
- 未验证项不按 0 分，给已知最低值—可能最高值。

## 等级

| 分数 | 等级 | 普通人解释 |
|---:|---|---|
| 0—1.0 | 很简单 | 导入或复制即可，几乎没有新增环境 |
| 1.1—2.5 | 简单 | 少量安装或账号配置，通常可独立完成 |
| 2.6—4.5 | 中等 | 多项依赖或系统差异，需要认真照文档操作 |
| 4.6—6.5 | 复杂 | 多工具、多服务或多账号，确认高频需求后再装 |
| 6.6—8.0 | 很复杂 | 接近生产流水线，需要维护和故障处理 |
| 8.1—10 | 系统级 | 接近采用一套完整应用或平台 |

## 额外涉及费用写法

报告中不用代码，只写以下自然语言之一：

- 无新增费用；
- 核心免费，扩展功能可能付费；
- 核心路径有一项必付或按量计费；
- 多项持续付费；
- 代码公开，暂未发现直接费用。使用许可未验证；
- 价格未验证。

仓库公开可读、可复制或未标价格，只能说明暂未发现直接收费。未找到明确许可证时，禁止写“核心免费”或“无新增费用”；必须用两个短句分别说明直接费用与使用许可。

## 报告要求

- 用户已拥有的环境可同时给首次采用分和按用户现状分；
- 额外涉及费用、Token、风险、许可证和质量不计入安装复杂度。
